# ProClinic — Data Models

## Entity Relationship Overview

```
CustomUser (accounts)
    │
    ├─[1:N]─► Patient ──────────┬─[1:N]─► Visit
    │           (patients)       ├─[1:N]─► LabReport
    │                            ├─[1:N]─► Appointment ──[0:1]─► Visit
    │                            ├─[1:N]─► Prescription ─[1:N]─► PrescriptionItem
    │                            └─[1:N]─► Invoice ──────[1:N]─► InvoiceItem
    │
    ├─[1:N]─► Publication (as doctor / approved_by)
    └─[1:N]─► AuditLog (as actor)
```

## Appointment (`appointments`)

Tracks every patient–doctor encounter slot with a full status lifecycle.

| Field | Type | Notes |
|---|---|---|
| `id` | AutoField | PK |
| `patient` | FK(Patient) | |
| `doctor` | FK(CustomUser) | Must have `role=DOCTOR` |
| `scheduled_time` | DateTimeField | |
| `reason` | TextField | Optional patient notes |
| `status` | CharField | See lifecycle below |
| `created_at` | DateTimeField | auto |
| `created_by` | FK(CustomUser) | Who booked it |
| `cancelled_at` | DateTimeField | Nullable |
| `cancelled_by` | FK(CustomUser) | Nullable |
| `cancellation_reason` | TextField | Optional |

**Status lifecycle:**
```
SCHEDULED → COMPLETED
          → CANCELLED  (via cancel())
          → RESCHEDULED (via reschedule())
          → NO_SHOW
```

**Model methods:**
- `cancel(user, reason='')` — sets status, records who/when/why
- `reschedule(new_time)` — updates `scheduled_time`, sets status to `RESCHEDULED`
- `is_cancellable` (property) — `True` only when `status == SCHEDULED`

|---|---|---|

- `approve(reviewer)` — sets status, records reviewer and timestamp

- `is_public` (property) — `True` when `status == APPROVED`

## AuditLog (`audit`)

Immutable log created automatically by Django signals.

| Field | Type | Notes |
|---|---|---|
| `id` | AutoField | PK |
| `actor` | FK(CustomUser) | Nullable — None for system/management commands |
| `action_type` | CharField | `CREATE` \| `UPDATE` \| `DELETE` \| `LOGIN` |
| `entity_type` | CharField | E.g. `"Patient"`, `"Appointment"` |
| `entity_id` | PositiveIntegerField | PK of the affected record |
| `timestamp` | DateTimeField | auto (`auto_now_add`) |
| `changes` | JSONField | CREATE: snapshot; UPDATE: `{field: {before, after}}`; DELETE: summary dict |

**Tracked entities:** Patient, Appointment, Prescription, Invoice, Publication, LabReport

**Never logged (sensitive):** `password`, `token`, `secret`, `api_key`, `access_token`, `refresh_token`  
**Logged as placeholder:** `pdf_file`, `image` fields → `"<file>"`
